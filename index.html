Ритмические конструкции

репчик состоит из строк (внезапно), их как правило 16 в куплете (спойлер: должно быть кратно 4)
строки должны соотноситься друг с другом или по количеству ударений (акцентный/тонический, стих)
или по количеству слогов (силлабический стих)
или по типу сочетаний ударных и безударных слогов (силлаботонический стих).
силлабический в русском почти не используется, потому что в отличие от пиндоского в слэйвянских языках у слов структура гораздо разнообразнее и значимость ударений выше,
но все равно за количеством слогов придется следить, чтоб не было внезапной тараторки

Способ с ударениями

метр - одна из схем звукового ритма, тупо чередование ударных и безударных слогов
основа бита - выбор bpm (удары в минутах если шо), по сути это и есть метр
то есть, орудуя ртом ты должен, используя инерцию метра,
наложить поверх живое ритмическое звучание.
Слушатели в свою очередь, воспринимая мембранами ушей этот ритмическое звучание,
интуитивно восстанавливают то метрическое задание,
которое ставил перед собой творец сея шедевра (происходит кач)

Некоторые commands

$ git log - чтобы check commits
$ git show ~первые символы идентификатора коммита~ // показывает, что появилось нового в коммите
$ git diff ~commit1~..~commit2~ //показывает разницу коммитов
$ git remote -v //check name remote repos and link
$ git status //check changes

Ветвление

$ git branch ~name~ //создание ветки (копии master)
$ git branch //show branches (активная помечена *)
$ git checkout ~name~ //переключение на ветку name
Если закоммитить изменения в ветку ~name~ и переключиться на master, то в файловой системе изменения не появятся.
Чтобы они появились, объединенияем ветки:
$ git merge ~name~ -m "your comments"//примененяем изменения из ветки name к основной версии и коммитим
$ git branch -d ~name~ //для удаления ветки
$ git fetch ~remote repos name~ ~branch name~ //pull без слияния(fetch+merge = pull)

Действия с коммитами

$ git checkout ~первые символы идентификатора коммита~ ~filename~
$ git commit --amend //fix last commit
$ git revert ~первые цифры id коммита(last commit - HEAD)~ // create commit that delete changes in ~name~ commit
